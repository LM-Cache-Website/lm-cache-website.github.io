动态后端管理

动态后端管理#

LMCache 提供了一组内部 API 端点，允许您在运行时 列出、关闭 和 创建 存储后端，而无需重启服务引擎。当您需要动态切换不同的存储配置时，这非常有用——例如，从 LocalDiskBackend 迁移到 GdsBackend，或将远程连接器从文件系统连接器更改为 Redis。

概述#

动态切换存储后端的工作流程是：

  1. **关闭**您想要替换的后端。

  2. **更新**相关配置通过 POST /conf API。

  3. 创建 新的后端 — 只有尚不存在的后端才会被创建。

任何**未**关闭的后端在创建过程中将被跳过，因此该操作是安全且幂等的。

API 端点#

GET /backends#

列出所有活动的存储后端。

curl http://localhost:7000/backends

响应：

{
  "LocalCPUBackend": "LocalCPUBackend",
  "RemoteBackend": "RemoteBackend"
}

DELETE /backends/{backend_name}#

关闭并移除特定的存储后端。此调用后，后端将完全关闭并从内部字典中移除，确保没有过时的引用残留。

curl -X DELETE http://localhost:7000/backends/RemoteBackend

响应：

{
  "status": "success",
  "message": "Backend RemoteBackend closed",
  "backends": {
    "LocalCPUBackend": "LocalCPUBackend"
  }
}

POST /backends#

根据当前的 LMCacheEngineConfig 创建新的存储后端。现有的后端将被跳过。

curl -X POST http://localhost:7000/backends

响应：

{
  "status": "success",
  "created": {
    "RemoteBackend": "RemoteBackend"
  },
  "backends": {
    "LocalCPUBackend": "LocalCPUBackend",
    "RemoteBackend": "RemoteBackend"
  }
}

用例示例#

从 LocalDiskBackend 切换到 GdsBackend#

如果您最初配置了本地磁盘后端，并希望在运行时迁移到 NVIDIA GPUDirect Storage (GDS)：

# 1. Close the old disk backend
curl -X DELETE http://localhost:7000/backends/LocalDiskBackend

# 2. Disable local_disk and set the GDS path
curl -X POST http://localhost:7000/conf \
  -H "Content-Type: application/json" \
  -d '{
    "local_disk": false,
    "gds_path": "/mnt/nvme/lmcache_gds"
  }'

# 3. Create the new GDS backend
curl -X POST http://localhost:7000/backends

# 4. Verify the new backend list
curl http://localhost:7000/backends

切换 RemoteBackend 连接器 (FS → Redis)#

要在不重启的情况下更改远程连接器类型：

# 1. Close the old remote backend
curl -X DELETE http://localhost:7000/backends/RemoteBackend

# 2. Update the remote URL to point to Redis
curl -X POST http://localhost:7000/conf \
  -H "Content-Type: application/json" \
  -d '{
    "remote_url": "redis://redis-host:6379"
  }'

# 3. Create the new remote backend with Redis connector
curl -X POST http://localhost:7000/backends

# 4. Verify
curl http://localhost:7000/backends

仅替换一个后端而保持其他后端不变#

如果只需要更新 RemoteBackend 而 LocalCPUBackend 应保持不变：

# Close only the remote backend
curl -X DELETE http://localhost:7000/backends/RemoteBackend

# Update config
curl -X POST http://localhost:7000/conf \
  -H "Content-Type: application/json" \
  -d '{"remote_url": "redis://new-redis:6379"}'

# Create — LocalCPUBackend is skipped (already present)
curl -X POST http://localhost:7000/backends

注意事项#

  • 关闭 LocalCPUBackend 是可能的，但应谨慎进行，因为许多其他后端依赖它作为中间缓冲区。

  • POST /backends 端点调用与引擎初始化期间使用的相同 CreateStorageBackends 工厂，因此支持所有后端类型（本地 CPU、本地磁盘、GDS、远程、P2P、插件等）。

  • 创建后端后，StorageManager 会自动刷新其内部引用（non_allocator_backends、local_cpu_backend 等）。
